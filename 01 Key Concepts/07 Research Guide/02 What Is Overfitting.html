Overfitting happens when the parameters of an algorithm are fine-tuned to fit the detail and noise of backtest data to the extent that it negatively impacts the performance of the algorithm on new data. This means that the noise or random fluctuations in the data are picked up by the algorithm and the trading parameters are structured to fit accordingly. The problem is that the parameters do not necessarily apply to new data and negatively impact the algorithm's ability to generalize.

Overfitting can manifest in a number of ways:

Data Practice
Data Dredging Performing many statistical tests on data and only paying attention to those that come back with significant results.
Hyper-Tuning Parameters Manually changing algorithm parameters to produce better results without altering the test data.
Stale Testing Data Not changing the backtesting data set when testing the algorithm. Any improvements might not be able to be generalized to different datasets.
Backfilling Data Introduces look-ahead bias. By default, LEAN uses forward-filled data.
Overfit Regression Models Overfitting regression, machine learning, or other statistical models will immediately introduce overfitting to an algorithm.

An aglorithm that is dynamic and generalizeable to new data will prove more valuable to funds and individual investors. It is more likely to survive across different market conditions and be applicable to new asset classes and markets.
