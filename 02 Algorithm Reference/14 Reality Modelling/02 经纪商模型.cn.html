我们提供一个快捷方式来设定我们支持的每个经纪商的常用模型和属性。这些经纪商模型会设定经纪商的手续费、订单成交模型、滑点模型和交易市场。另外它们会验证能否向经纪商提交交易（例如，将股票交易订单提交给外汇经纪商）。

经纪商模型会设定：

  • 手续费
  • 经纪商支持的资产类型
  • 检验订单类型和设置
  • 默认账户类型（保证金或现金）
  • 是否支持盘后交易
  • 资产的默认杠杆
  • 默认的结算模式

这使您可以很好的控制算法行为，并允许您模拟任何经纪商。

除了我们的默认经纪商模型（默认经纪商模型），我们为Interactive Brokers（InteractiveBrokersBrokerageModel ）、FXCM（FxcmBrokerageModel）、OANDA（OandaBrokerageModel）和Coinbase Pro（GDAX）提供经纪商模型。

经纪商模型将覆盖您给某支证券设置的其他模型。 

// 用辅助方法设置经纪商模型：
SetBrokerageModel(BrokerageName.FxcmBrokerage); // 保证金账户的默认值
SetBrokerageModel(BrokerageName.GDAX, AccountType.Cash); //或者覆盖账户类型

// 支持的经纪商名：
BrokerageName.FxcmBrokerage
             .OandaBrokerage
             .InteractiveBrokersBrokerage
             .GDAX

// 您也可以自定义经纪商模型： IBrokerageModel
class MyBrokerage: DefaultBrokerage {
   // 在此自定义经纪商
} 

SetBrokerageModel(new MyBrokerage());

# 用辅助方法设置经纪商模型：
self.SetBrokerageModel(BrokerageName.FxcmBrokerage) # 保证金账户的默认值
self.SetBrokerageModel(BrokerageName.GDAX, \ 
                       AccountType.Cash) # 或者覆盖账户类型

# 支持的经纪商名：
BrokerageName.FxcmBrokerage
             .OandaBrokerage
             .InteractiveBrokersBrokerage
             .GDAX
