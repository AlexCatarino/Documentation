When history requests for are for a known set of Symbols QuantConnect returns a multi-index Data Frame with all the historical results.

The simplest form of history request is for one security, where we return an array of security data. If you do not specify a resolution QuantConnect defaults to the resolution of the security in your algorithm. This API follows the pattern: self.History[Type]( symbol, bar_count, resolution = null) History<Type>(Symbol symbol, int barCount, Resolution resolution = null) 

# Requesting By Bar Count: 14 IBM TradeBars 

self.bars = self.History("IBM", 14) 
self.bars = self.History[TradeBar]("IBM", 14)
self.bars = self.History[TradeBar]("IBM", 14, Resolution.Minute)


# Requestion By Period: 2 Weeks Of IBM Data

self.bars = self.History("IBM", timedelta(14))
self.bars = self.History("IBM", timedelta(14), Resolution.Minute)

// Requesting By Bar Count: 14 IBM TradeBars
var bars = History<TradeBar>("IBM", 14);
var bars = History<TradeBar>("IBM", 14, Resolution.Minute);

// Requestion By Period: 2 Weeks Of IBM Data
var bars = History<TradeBar>("IBM", TimeSpan.FromDays(14));
var bars = History<TradeBar>("IBM", TimeSpan.FromDays(14), Resolution.Minute);
