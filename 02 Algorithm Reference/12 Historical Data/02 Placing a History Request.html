The QuantConnect historical data API has many different options to give you the greatest flexibility in how to apply it to your algorithm. Over the years it has evolved to handle different data formats, data resolutions and use cases, but we have always strived to keep two constants to its design:

History Key Concept #1:
  • History requests can be done by a bar count, or,
  • History requests can be done by a period of time.

Below we'll layout every API endpoint and how follows these two options. Secondly, as we've added new security classes we've had to support new data formats as such some assets return QuoteBars, and other assets return TradeBars. When placing a history request its important to consider what data will be returned.

History Key Concept #2:
  • History requests without a security specified return data as a Slice object.
  • Single security history requests return data in the data format of the security.
  • Multi-security (batch) requests return data as a data frame.
  • Multi-security (batch) requests return data as a Slice object.
