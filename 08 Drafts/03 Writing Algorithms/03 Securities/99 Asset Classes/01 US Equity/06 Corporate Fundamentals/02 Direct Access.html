To get fundamental data, access the Fundamental properties in your fundamental universe selection function or access the Fundamentals property of the Equity objects in your fundamental universe. 

var fundamentals = Securities[_symbol].Fundamentals;
fundamentals = self.Securities[self.symbol].Fundamentals

Without an active subscription.
QCAlgorithm.Fundamentals returns Fundamental for a single security or a list depending on the input

// Single asset 
var ibm = QuantConnect.Symbol.Create("IBM", SecurityType.Equity, Market.USA);
var ibmFundamental = Fundamentals(ibm);

// Multiple assets
var nb = QuantConnect.Symbol.Create("NB", SecurityType.Equity, Market.USA);
var fundamentals = Fundamentals(new List{ nb, ibm }).ToList();

# Single asset
ibm = Symbol.Create("IBM", SecurityType.Equity, Market.USA)
ibmFundamental = self.Fundamentals(ibm)

# Multiple assets
nb = Symbol.Create("NB", SecurityType.Equity, Market.USA)
fundamentals = self.Fundamentals([ nb, ibm ])

H4 : Has Fundamental Data
- Check if it has fundamental data
- Fundamentals property.HasFundamentalData property is false if the Equity doesn't have this data (ETF)

var hasFundamentalData = Securities["SPY"].Fundamentals.HasFundamentalData;
has_fundamental_data = self.Securities["SPY"].Fundamentals.HasFundamentalData

H4 : Saving Reference
- We cannot keep a reference of sub classes (eg Fundamentals.EarningRatios) since the object is reused
- We can keep a reference of Fundamentals:

_fundamentals = Securities["SPY"].Fundamentals;
var earningRatios = _fundamentals.EarningRatios;
    
// WARNING: Don't save a reference to the properties of the Fundamentals object
// _earningRatios = Securities["SPY"].Fundamentals.EarningRatios;
self.fundamentals  = self.Securities["SPY"].Fundamentals
earning_ratios = self.fundamentals.EarningRatios
