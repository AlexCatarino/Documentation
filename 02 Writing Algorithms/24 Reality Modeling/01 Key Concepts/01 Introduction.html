What are models for? Why use them for your backtesting? What is reality modeling: - In backtesting, the strategy uses historical data to simulate trading decisions that are submited to market conditions. - We try to make it as realistic as possible so the strategy performs as it would have in real-time - We provide basic default models that assume you are trading on highly liquid assets, but if you are trading high volumes or on low volume assets, you should update these models to be more realistic. Why is there reality modeling: - We try to make it as realistic as possible so the strategy performs as it would have in real-time - We need to model the behavior of the portfolio, brokerage, fills, slippage, options, and capacity - Models can be used to improve the accuracy of your backtesting

QuantConnect endeavors to make our backtesting as realistic as possible by providing high-resolution data, spread information, and powerful transaction models. These models are customizable on a per security basis through setters on the API.

Models can be used to improve the accuracy of your backtesting. We provide basic default models that assume you are trading on highly liquid assets, but if you are trading high volumes or on low volume assets, you should update these models to be more realistic.

All models are set on a per security basis. To set a model, first fetch the security object and apply your model. 

//Set IBM to have a constant $1 transaction fee. 
Securities["IBM"].FeeModel = new ConstantFeeTransactionModel(1);

All models should be set up in your Initialize() method.
