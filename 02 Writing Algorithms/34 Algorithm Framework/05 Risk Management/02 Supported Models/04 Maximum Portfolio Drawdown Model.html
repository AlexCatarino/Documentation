The MaximumDrawdownPercentPortfolio model monitors the portfolio drawdown. The drawdown can be relative to the starting portfolio value or the maximum portfolio value. When the portfolio value drops below a percentage threshold, the model liquidates the portfolio. After the portfolio is liquidated, the model resets.

The model accepts the following arguments:
-maximumDrawdownPercent: The maximum percentage drawdown allowed for algorithm portfolio compared with starting value; default 0.05 (5%)
-isTrailing: If "false", the drawdown will be relative to the starting value of the portfolio. If "true", the drawdown will be relative the last maximum portfolio value; default False 

AddRiskManagement( new MaximumDrawdownPercentPerSecurity(maximumDrawdownPercent, isTrailing) );
self.AddRiskManagement( MaximumDrawdownPercentPerSecurity(maximum_drawdown_percent, is_trailing) )
