You can set a warm up period based on a number of bars or a period of time. To set a warm up, in your algorithm's Initialize method, call the SetWarmUp method.

// Wind time back 7 days from start
SetWarmUp(TimeSpan.FromDays(7));

// Feed in data for 100 trading days before the start date
SetWarmUp(100, Resolution.Daily);

// Feed in 100 bars before start date
SetWarmUp(100);

# Wind time back 7 days from start
self.SetWarmUp(timedelta(7))

# Feed in data for 100 trading days before the start date
self.SetWarmUp(100, Resolution.Daily)

# Feed in 100 bars before start date
self.SetWarmUp(100)

If you pass an integer and a resolution to the SetWarmUp method, LEAN calculates the start of the warm up period from the arguments. If you call self.SetWarmUp(100, Resolution.Daily)self.SetWarmUp(100, Resolution.Daily), the warm up period consists of 100 trading days. In this example, if your algorithm has a security subscription for daily data, the warm up data for the security is 100 daily bars. If your algorithm has a security subscription for minute resolution data, the warm up data for the security is 100 days worth of minute bars.

If you call SetWarmUp with just an integer argument, the resolution argument is set to the highest resolution in your universe. In the following example, the warm up period is 100 minutes worth of data. If your algorithm doesn't have any data subscriptions, the resolution argument is set to the UniverseSettings.Resolution.

AddEquity("SPY", Resolution.Minute);
AddCrypto("BTCUSD", Resolution.Second);
SetWarmUp(100);

self.AddEquity("SPY", Resolution.Minute)
self.AddCrypto("BTCUSD", Resolution.Second)
self.SetWarmUp(100)
