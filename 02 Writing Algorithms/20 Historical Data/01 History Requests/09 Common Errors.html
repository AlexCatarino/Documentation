Errors can occur when you request historical data.

Empty DataFrame Errors

If the history request returns an empty DataFrame and you try to slice it, an exception is thrown. To avoid issues, check if the DataFrame contains data before slicing it.

df = self.History(symbol, 10).close    # raises exception if the request is empty

def GetSafeHistoryCloses(symbols):
    if not symbols:
        self.Log(f'No symbols on {self.Time}')
        return  False, None
    df = self.History(symbols, 100, Resolution.Daily)
    if df.empty:
        self.Log(f'Empy history for {symbols} on {self.Time}')
        return  False, None
     return True, df.close.unstack(0)

Numerical Precision Errors

Some factor files have INF split values, indicating that the stock has so many splits that prices can't be calculated with correct numerical precision. To allow history requests with these symbols, we need to move the starting date forward when reading the data. If there are numerical precision errors in the factor files for a security in your history request, LEAN throws the following error:

"Warning: when performing history requests, the start date will be adjusted if there are numerical precision errors in the factor files."
