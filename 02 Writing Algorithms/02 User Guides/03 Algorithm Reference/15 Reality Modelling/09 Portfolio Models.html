Portfolio models control how order fills are applied to your portfolio. They take an $[OrderEvent,T:QuantConnect.Orders.OrderEvent], $[Security,T:QuantConnect.Securities.Security], and $[SecurityPortfolioManager,T:QuantConnect.Securities.SecurityPortfolioManager] object and update the holdings to reflect the new final position. You should only need to update your portfolio model when you are creating a new asset type.

Portfolio models implement the $[ISecurityPortfolioModel,T:QuantConnect.Securities.ISecurityPortfolioModel] interface.
