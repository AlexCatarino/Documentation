What is look-ahead bias:
-The practice of making decisions using information that would not be available until some time in the future. (from Glossary)

Why is look-ahead bias bad:
- If look-ahead bias seeps into your algorithm, it will perform differently in live trading and backtesting
- In backtesting, you'll receive the data earlier than it would actually be available in reality

How to avoid look-ahead bias:
-The data points are injected into your algorithm at the EndTime (Time + Period) that you set in the Reader method.
-You should set the EndTime to the point of time when the data point is actually available.
-A lot of external sources apply timestamps to data differently than we do. For instance, on some platforms, daily bars are displayed with the date that they opened. We display daily bars with the date they closed. If you set the EndTime to the start of the bar, you'll receive the bar earlier in backtests than you would in live trading
