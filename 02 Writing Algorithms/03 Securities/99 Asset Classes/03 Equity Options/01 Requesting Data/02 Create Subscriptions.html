What:
- Create a security subscription

Why:
-To add a data feed for the security to your algorithm
   - To enable trading / fills / quotes
   - To receive data updates in your algorithm + portfolio
   - To make trading decisions
-This adds the security to your "user-defined" / static universe

How:

because the following techniques are "universes"

            - You create a equity subscription
            - Create an option subscription using the equity ticker
            - Can provide a contract selection method
                 - All the universe selection options we have
            - Creates Option contracts
            - Creates Option chain

- Technique 1: OptionChainProvider
    - https://www.quantconnect.com/docs/data-library/options#Options-Requesting-Options-Data---OptionChainProvider
    - If you add an option in the current slice, you'll need to wait until the next slice to receive data and trade the option. You can override this with by setting the current price of the option with a security initializer
     -Instead of subscribing to the entire chain at every timestep, you can use OptionChainProvider/FutureChainProvider and AddOptionContract/AddFutureContract
    - If you use AddOptionContract, you must manually set the underlying volatility model. (https://github.com/QuantConnect/Documentation/issues/102)
    - Need to wait 1 timestep, or you can create a security initializer to set the last price

- For a universe, see Universes > Equity Options

-To remove a subscription, use RemoveSecurity() / RemoveOptionContract()
