What:
- Create a security subscription

Why:
-To add a data feed for the security to your algorithm
  - To enable trading / fills / quotes
  - To receive data updates in your algorithm + portfolio
  - To make trading decisions
-This adds the security to your "user-defined"

How:
Technique 1: AddFuture + AddFutureOptionContract
-The Future contract is the Underlying
-For a list of properties you can filter Future contracts by, see Futures > Requesting Data.
-For a list of properties you can filter FOP contracts by, see Data Formats > Option Contracts.


Technique 2: FutureChainProvider + AddFutureContract + OptionChainProvider + AddOptionContract
example: https://github.com/QuantConnect/Lean/blob/master/Algorithm.Python/FutureOptionPutOTMExpiryRegressionAlgorithm.py
-Faster backtesting because fewer subscriptions

-To remove a subscription, use RemoveSecurity()/RemoveOptionContract

-Alternatively, define a FutureOptions universe: https://www.quantconnect.com/docs/v2/writing-algorithms/universes/future-options
