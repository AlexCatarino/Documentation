AddFuture

AddFuture(
 String ticker,
 Resolution resolution,
 String market,
 Boolean fillDataForward,
 Decimal leverage,
 DataMappingMode DataMappingMode,
 DataNormalizationMode DataNormalizationMode,
 Int32 contractDepthOffset
)

Creates and adds a new Future security to the algorithm

Parameters

Type Name Description
StringtickerThe future ticker.
ResolutionresolutionThe Resolution of market data, Tick, Second, Minute, Hour, or Daily. Default is Minute.
StringmarketThe futures market, Market. Default is value None and looked up using BrokerageModel.DefaultMarkets in AddSecurity``1.
BooleanfillDataForwardIf True, returns the last available data even if none in that timeslice. Default is True.
DecimalleverageThe requested leverage for this equity. Default is set by SecurityInitializer.
DataMappingModeDataMappingModeContinuous contracts mapping modes.Options: ['LastTradingDay', 'FirstDayMonth', 'OpenInterest']
DataNormalizationModeDataNormalizationModeSpecifies how data is normalized before being sent into an algorithm.Options: ['Raw', 'Adjusted', 'SplitAdjusted', 'TotalReturn', 'ForwardPanamaCanal', 'BackwardsPanamaCanal', 'BackwardsRatio']
Int32contractDepthOffsetThe continuous future contract desired offset from the current front month. For example, 0 (default) will use the front month, 1 will use the back month contract.

Returns

Future - The new security
