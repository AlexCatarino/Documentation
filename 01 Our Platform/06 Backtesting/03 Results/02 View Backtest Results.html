The content in the backtest results page is updated as your backtest executes. You can close or refresh the window without interrupting the backtesting node because the node is processing on our servers. Unless you explicitly make the backtest public, only you can view your backtest results. The following table describes the functionality of each tab at the bottom of the backtest results page:

Tab Description
Overview Various statistics on the backtest performance.
Report Create a backtest report.
Orders A table of orders that occurred in the backtest.
Logs The logs generated through the backtesting, including error messages, debug statements, quit statements.
Code The project files that were used to deploy the backtest.
Share Make the backtest public and generate links to share the results.

We give an arbitrary name (for example, "Smooth Apricot Chicken") to your backtest result files, but you can rename them. If you delete a backtest result or you are inactive for 12 months, we archive your backtest results. If you share a backtest result, it's public domain and always stored.
