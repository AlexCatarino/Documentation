Breakpoints are lines in your algorithm where execution pauses. You need atleast one breakpoint in your code files to start the debugger. To add a breakpoint, open a project and then click to the left of a line number.

To customize what happens when a breakpoint, right-click the breakpoint and then select one of the following options from the drop-down menu:

Option Additional Steps Description
Expression Enter an expression and then press Enter. When the expression is true, the algorithm breaks.
Hit Count Enter an integer and then press Enter. When the breakpoint is hit the number of times you specify, the algorithm breaks.
Log Message Enter a message and then press Enter. When the breakpoint is hit, the message is added to the log file. Expressions within { } are interpolated.
