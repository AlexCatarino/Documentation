When all of the QuantConnect members report the data issues that they find, we can ensure the datasets are high quality for everyone. The easier it is for our Data Team to detect and reproduce the issue that's reported, the faster the issue is resolved. To ensure we can detect and reproduce the data issues that are reported, the reports should include the following information:

  • Type of data issue (for example, a delisting error, name mapping error, or missing data points)
  • Security ticker
  • Start and end times
  • Security type
  • Market
  • Resolution
  • Description
  • Attached backtest or research notebook that demonstrates the issue

If you find an issue with the historical data in a dataset, report it in the Dataset Market. If you encounter an issue with live data, email us a description of the issue.
