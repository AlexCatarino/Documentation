When all of the QuantConnect members report the data issues they find, we can ensure the datasets are high quality for everyone. The easier it is for our Data Team to detect and reproduce the issue that's reported, the faster the issue is resolved. Data issue reports should include the following information:

  • Type of data issue (for example, a delisting error, name mapping error, or missing data points)
  • Security ticker
  • Start and end times
  • Security type
  • Market
  • Resolution
  • Description
  • Attached backtest or research notebook that demonstrates the issue

The data issue reporting area in the community forum includes a form so that you can provide the information above. If you encounter an issue with live data, report the issue to support@quantconnect.com instead of opening a data issue.
