What are code settings:
- Features in the IDE that can be enabled/disabled

Why are there code settings:
- So you can customize the functionality of the IDE to fit your preferences
- They are saved to your account, so you don't have to adjust the settings every time you log in from a different device

Strategy Builder

What is the Strategy Builder:
- Redirect to Projects > IDE > Strategy Builder docs

What is the Strategy Builder setting:
- Allows you to opt-in or opt-out of the Strategy Builder.
- If you opt-out of the Strategy Builder, it won't be displayed when you create a new project

Why enable the Strategy Builder:
- If you're new to the platform, the Strategy Builder provides an easy way to get started building framework algorithms without writing any code.

Why disable the Strategy Builder:
- If you don't use the Strategy Builder, disable it so that new projects are created faster. You won't have to exit the Strategy Builder every time.

Automatic Builds

What are automatic builds:
- With automatic builds, the project you're working on will be built after every keystroke you make

Why enable automatic builds:
- You'll know if you've made a coding error right away, even before you go to manually build the project and run the backtest.
- It will save you time from having to build the project before running a backtest

Why disable automatic builds:
- Personal preferences
     - You may not want to see red lines & pop-ups in the IDE before you've finished writing an entire line of code.
- The pop-ups can be distracting

Code Completion

What is code completion:
- GUI tool that shows options for auto-complete and presents the members that are accessible from the current object
- Redirect to Projects > IDE > Autocomplete and Intellisense docs

Why enable code completion:
- To speed up your development
- To reduce the number of times you need to use the API reference or API tree.

Why disable code completion:
- You may find the pop-ups distracting

Parameter Highlighting

What is parameter highlighting:
- Hard-coded parameter values are highlighted in the IDE

Why enable parameter highlighting:
- The IDE will be able to show where parameters were involved in the algorithm, giving you an idea of if the algorithm could be over-fit and pointing out locations where the algorithm could be made more general.
- Being notified of when you've added a parameter to your algorithm can help you develop more general algorithms with fewer parameters.

Why disable parameter highlighting:
- You find the pop-ups distracting

Meta-Data Sharing

What is meta-data:
- The backtest meta-data would include performance statistics, significant holdings, and the end of day equity curves. It would be just enough information to determine if the fund is interested to see it tested live in Alpha Streams. As always, no one can see your source-code and you own your algorithm intellectual property.

Why enable meta-data sharing:
- If we can see that your backtests have good results, you may be contacted for licensing opportunities before submiting an Alpha Streams algorithm

Why disable meta-data sharing:
- If you prefer to keep all of your backtest information private, you have the option to opt-out of meta-data sharing.
