There are limits on the number of concurrent backtests that the optimizer can run and the number of parameters that can be optimized.

Number of Concurrent Backtests

The optimizer can run up to 24 concurrent backtests.

Number of Parameters

The cloud optimizer currently only supports optimizing up to 2 parameters. The number of parameters is limited because grid search is the only supported optimization strategy and, since grid search tests all of the parameter combinations, optimizing more than 2 parameters requires too many backtests. When the optimizer has more efficient optimization strategies, we can alleviate this limitation. To optimize more parameters before we add more optimization strategies, use the CLI.
