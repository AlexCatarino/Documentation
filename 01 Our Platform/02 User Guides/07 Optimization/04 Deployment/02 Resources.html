The optimization nodes that backtest your algorithm are not the nodes to which your organization subscribes. The optimization nodes are a custer of nodes that exclusively run optimization jobs. You can use 1-24 nodes when running optimization jobs. If you use multiple nodes, the backtests run concurrently and the nodes all of have the same specifications. The following table shows the specifications of the optimization node models:

Node Type Number of Cores RAM
O2-8 2 Cores 8 GB
O4-12 4 Cores 12 GB
O8-16 8 Cores 16 GB

Select the optimization node that is most appropriate for your trading algorithm:

  • O2-8: Relatively simple strategies with less than 100 assets
  • O4-12: Strategies with less than 500 assets and simple universe selections
  • O8-16: Complex strategies and machine learning
