What is the cost:
-Optimization is paid for in QCC from your organization
-The cost is calculated after you run a backtest for the strategy
-Using the last run backtest, the optimization search model selected and the number of parameters, QuantConnect can create a rough estimate of the total time it will take to run this optimization. You can speed up this optimization by assigning it more processing nodes, or depending on your algorithm making sure it has sufficient CPU available. Note: There is a spin up time of roughly 15s on each backtest so even very short backtests can sometimes take some time to process.
-Optimization nodes are rented on a time-basis
-Using multiple nodes makes the optimization run faster but doesn't really cost more because each of the nodes are used for a shorter period of time
    - It may cost more when you factor in the spin-up time of each of the nodes
-The cost is calculated based on the time that the last backtest took to complete. Perhaps the parameters that were used in that backtest were favorable for speedy execution, while some extreme combinations led to a slow one. For example, one combination could result in a handful of trades (e.g. moving average cross with very different lookback periods leading to rare cross events) while another combination could produce one trade per minute (e.g. moving average cross with very close lookback periods, leading to a lot of crosses).

Why do I have to pay:
- QC provides convenience by providing the hardware and platform
- If you have your own hardware, you can run local optimizations with local data and local hardware without paying QCC
