What are delistings:
-When an alpha is removed from the marketplace
-If an algorithm does not meet the live performance requirements, it’s delisted

What would cause an alpha to be delisted:
-3 month grace period
-https://www.quantconnect.com/docs/key-concepts/reconciliation#Reconciliation-Scoring-Reconciliation
-<40% DTW  or  return correlation > .5  or  Live sharpe > -3
      - DTW of 0 is the returns are the same between backtesting and live
      - Return correlation should be as close to 1 as possible
-Logic error and the author doesn’t fix it
      - We need to temporarily hide the alpha from the marketplace if the author doesn’t fix the error quickly
-Run-time errors and the author doesn’t fix it
      - QuantConnect contacts alpha authors to help them fix run-time errors to keep the alpha in the market
      - if there is an error in the code and the author doesn’t address it (like SetHoldings in a crypto algorithm), QC will try contacting the author

What if people are invested in an alpha that passes the delisting metrics:
-If there are people invested in the alphas, we won’t delist it, but we can talk to the investors.
