What is concurrent backtesting:
-Running multiple backtests in your organization at the same time
-You need multiple backtest nodes to run concurrent backtests

Why would I want concurrent backtesting:
-To run multiple backtests at once
-To speed up your strategy development
-To have multiple team members within your organization running backtests at the same time
-You can run optimization to run concurrent backtests, but those are the same strategy and aren't your org's nodes

Why would want multiple nodes:
-More nodes = more concurrent backtests
