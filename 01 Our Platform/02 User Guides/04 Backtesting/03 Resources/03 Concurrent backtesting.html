-To run concurrent backtests, you need multiple backtesting nodes
-One node can run one backtest at a time
-More nodes = more concurrent backtests
-You can run optimization to run concurrent backtests, but those are the same strategy and aren't your org's nodes


<h3>Limitations
- To run backtests, you need to have an available backtesting node in your organization
- The amount of backtesting nodes you can have in your organization depends on the organization's tier
    - Free => 1
    - Quant Researcher => 2
    - Team => 10
    - Trading Firm & Institution => Unlimited
