The content in the backtest results page is updated as the backtest executes. You can close or refresh the window without interrupting the backtesting node because the node is processing on our servers. Only you can view your backtest results unless you explicitly make the backtest publish and share the results. The backtest results page has a table at the bottom of the page with the following tabs:

  • Overview: Various statistics on the backtest performance
  • Report: Create a backtest report
  • Orders: A table of orders that occurred in the backtest.
  • Logs: The logs generated through the backtesting, including error messages, debug statements, quit statements.
  • Code: The project files used to deploy the backtest.
  • Share: Make the backtest publish and generate links to share the results.

Backtests result files are given an arbitrary name when the backtest is deployed. However, you can rename the backtest result files using the IDE. Backtest results are archived after 12 months of inactivity or immediately on deletion. If you share a backtest result, it's public domain and always stored.
