Organizations can lease resource nodes for backtesting, research, and live trading. The content below outlines the purpose and specifications of each of the available resource nodes.

Backtesting nodes

A backtesting node is needed for backtesting trading strategies. Adding more backtesting nodes allows you to run more concurrent backtests. Depending on the design of your algorithm, you can add more powerful nodes to run faster backtests. Nodes with more RAM allocation can handle more intensive operations such as machine learning and options data processing. Here are the specifications of each backtesting node:

Node Type Number of Cores Processing Speed RAM
B-MICRO 1 Cores 3.7 Ghz 8 GB
B2-8 2 Cores 4.9 Ghz 8 GB
B4-12 4 Cores 4.9 Ghz 12 GB
B8-16 8 Cores 4.9 Ghz 16 GB

Each organization gets one free B-MICRO backtesting node. However, this node is replaced when the organization adds another backtesting node to their subscription. The B-MICRO backtesting node incurs a 20 second delay when running backtests.

Research nodes

A research node is needed to use the research environment. More powerful research nodes allow you to handle more data and run faster computations in your notebooks. Here are the specifications of each research node:

Node Type Number of Cores Processing Speed RAM
R1-4 1 Core 2.4 GHz 4 GB
R2-8 2 Cores 2.4 Ghz 8 GB
R4-12 4 Cores 2.4 GHz 12 GB
R8-16 8 Cores 2.4 GHz 16 GB

Each organization gets one free R1-4 research node. However, this node is replaced when the organization adds another research node to their subscription.

Live trading nodes

A live trading node is needed for each live algorithm the organization deploys. More powerful live trading nodes allow you to run algorithms with larger universes, increase your notifications limit, and train machine learning models for a longer period of time. Here are the specifications of each live trading node:

Node Type Number of Cores Processing Speed RAM
L-MICRO 1 Core 2.7 Ghz 0.5 GB
L1-1 1 Core 2.7 GHz 1 GB
L1-2 1 Core 2.7 GHz 2 GB
L1-4 1 Core 2.7 GHz 4 GB

Customizations

Each resource node has a default name assigned to it when it’s initially added to an organization. The default name includes the node type and an arbitrary string of characters. However, resource nodes can be renamed to suit the organization’s needs.
