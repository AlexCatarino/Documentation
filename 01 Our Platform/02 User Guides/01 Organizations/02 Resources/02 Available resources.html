What hardware resources are available:
-Organizations can lease resource nodes for backtesting, research, and live trading.

What are the specs of each hardware node type:
   - <h4>Backtesting nodes
          - needed for backtesting trading strategies
          - more backtesting nodes allows you to run more concurrent backtests
          - more powerful nodes to run faster backtests
          - more RAM allocation can handle more intensive operations (ML, Options, large universe)
          - <Table of node specs>
          - one free B-MICRO backtesting node.
                -replaced when the organization adds another backtesting node to their subscription
                -20 second backtest delay
   - <h4>Research nodes
          - needed to use the research environment
          - more powerful research nodes => handle more data, run faster computations
          - <Table of node specs>
          - one free R1-4 research node
                - replaced when the organization adds another research node to their subscription
   - <h4>Live trading nodes
          -needed for each live algorithm the organization deploys
          -deployed to out co-located servers
          -more powerful nodes => larger universes, more live notifications, longer ML training periods
          -<Table of node specs>

What is the naming scheme of hardware resources:
-default name assigned
     - Includes node type and an arbitrary string of characters
     - Can rename
