Our platform supports using machine learning techniques in your trading strategies. The ObjectStore provides a place for storing and retrieving machine learning models. By using the ObjectStore, you can train models in the research environment before loading them into the backtesting and live trading environment. Additionally, you can store models in the ObjectStore to save you from having to retrain the model between backtests and live deployments.

Saving and loading models with pickle

Most machine learning models can be saved to the ObjectStore by serializing the model with pickle and then using the SaveBytes method. Loading the model is then done by retrieving the bytes from the ObjectStore with ReadBytes, converting the bytes into a bytearray, and then loading the model with pickle. The snippet below demonstrates the process of saving and loading models with pickle serialization. For a full example, refer to this notebook.

import pickle

# Save the model
self.ObjectStore.SaveBytes('model_key', pickle.dumps(model))

# Load the model
model_bytes = self.ObjectStore.ReadBytes('model_key')
model = pickle.loads(bytearray(model_bytes))

Saving and loading keras models

To save a keras model to the ObjectStore, use the serialize_keras_object method from the keras.util library to convert the model into a JSON object. The json.dumps method can then convert the JSON object into a string that can be saved in the ObjectStore. Once saved, Keras models can be loaded from the ObjectStore and reconstructed by using the json.loads and Sequential.from_config methods. The snippet below demonstrates the process of saving and loading keras models. For a full example, refer to this research notebook.

import json
from keras.utils.generic_utils import serialize_keras_object

# Saving the model
model_str = json.dumps(serialize_keras_object(model))
self.ObjectStore.Save('model_key', model_str)
