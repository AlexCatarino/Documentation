The optimization nodes that backtest your algorithm are not the backtesting nodes to which your organization subscribes. The optimization nodes are a cluster of nodes that exclusively run optimization jobs. You can use 1-24 nodes when running optimization jobs. If you use multiple nodes, the backtests run concurrently and all of the nodes have the same specifications. The following table shows the specifications of the optimization node models:

Model Number of Cores RAM (GB)
O2-8 2 8
O4-12 4 12
O8-16 8 16

Select the optimization node that is most appropriate for your trading algorithm:

  • O2-8: Relatively simple strategies with less than 100 assets
  • O4-12: Strategies with less than 500 assets and simple universe selections
  • O8-16: Complex strategies and machine learning
