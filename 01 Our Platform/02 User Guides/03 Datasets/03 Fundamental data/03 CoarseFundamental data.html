To access fundamental data in your algorithms, you first need coarse universe selection. The coarse universe selection method you define accepts CoarseFundamental objects and returns a list of Symbol objects. The CoarseFundmental objects are updated once per day. CoarseFundamental objects contain the following data attributes for each Equity asset:

  • Market - Market location
  • DollarVolume - Dollar volume of the previous day
  • Volume - Volume of the previous day
  • HasFundamentalData - True if the security has fundamental data
  • PriceFactor - The price factor for the given date
  • SplitFactor - The split factor for the given date
  • PriceScaleFactor - The combined factor used to create adjusted prices from raw prices
  • Symbol - Symbol representation for the underlying Security
  • Price - Yesterday’s closing price
  • AdjustedPrice - Yesterday’s split- and dividend-adjusted closing price
  • IsFillForward - True if this piece of data is filled forward

To include CoarseFundamental objects in your algorithms, you need the US Coarse Universe Constituents dataset. This dataset is a daily universe of all trading stocks in the US for a given day, with the end of day price and volume. The data covers 30,000 US Equities in total, with approximately 8,000 equities per day. The data starts in January 1998 and is delivered each trading day. This dataset is created by taking the closing auction price tick from the daily L1 trade and quote exchange dumps.
