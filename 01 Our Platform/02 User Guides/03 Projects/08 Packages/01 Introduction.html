What are packages:
    - Third-party code files that can be imported to your algorithm for use in backtesting, live trading, and research
    - These provide additional functionality than the classes/methods you define yourself
    - A package is a collection of modules
            - "A module is a file containing Python definitions and statements." // from python docs

Why use packages:
     - It's faster to use a pre-built, open source package than to write the functionality yourself
     - Speeds up your development process
     - Match the industry-standard -- there are many packages commonly used by firms to develop quant trading software/strategies

How(?):
     -To use a package, add the relevant using/import statement to your program
     - a module is only loaded once per interpreter session to promote efficiency
