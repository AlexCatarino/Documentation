Projects contains code files (.py or .cs) and notebook files (.ipynb). Run backtests with code files and launch the Research Environment with notebook files. Code files and notebook files can import code from other code files. However, notebook files use individual research nodes, so they can't import code from other notebook files. You can rename, move, and delete files in the web IDE to aid navigation. When you run backtests, your project's code files must subclass the QCAlgorithm class so that you can plug into the Lean engine and execute the backtest.
