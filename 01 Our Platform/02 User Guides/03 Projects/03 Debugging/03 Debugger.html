What is the debugger:
    - Built-in tool to aid debugging by enabling you to set breakpoints, observe variables, step through your program
    - You can control the steps, place breakpoints, what variables to watch
     - Can set breakpoints
          - A point where you want your algorithm to pause
     - Can watch variable values
          - These are variables you want to track. For ex: `self.ema.Current.Value`
          - These show the variable value at the current point in the algorithm
          - Hover over the variable value for easier viewing (Dataframes are displayed nice)
     - Can control algorithm stepping
          - Play button - continue execution until next break point
          - Stop button - exit debugger
          - arrow button - step to the next line of code

Why use it:
    - To correct coding errors in a project
    - Options to use local IDE debugging through CLI
    - If using the CLI, you can setup local debugging with several IDEs (link to CLI docs)
