Coding errors are errors that cause the project to not build, throw exceptions, or behave unexpectedly. There are generally 3 types of coding errors: build, runtime, and logic errors. Each type of error occurs for different reasons, as outlined below.

Build Errors

Build errors occur when the interpreter's syntax check fails. An example code snippet that produces a build error is the following:

var a = 1;
if (a = 2) {}
    
a = 1
if a = 2:
    pass

When build errors occur, you're notified in the following ways:

  • The number of build errors is displayed on the Build button.
  • A red underline is applied to the line with the error.
  • A warning message is added to the Console.

Runtime Errors

Runtime Errors, also called exceptions, occur when the interpreterâ€™s syntax checks pass but an error occurs during execution. An example code snippet that produces a runtime error is the following:

var a = new List() {1};
var b = a[1];
    
a = [1]
a[1]

When runtime errors occur, a stack trace of the error is added to the Console and the log file. For example, the snippet above produces the following error message:

Runtime Error: IndexError : list index out of range
  at OnData
    a[1]
===
   at Python.Runtime.PyObject.Invoke(PyTuple args in main.py: line 17
     (Open Stacktrace)
Runtime Error: Index was out of range. Must be non-negative and less than the size of the collection. (Parameter 'index') in Main.cs:line 24 (Open Stacktrace)

The stack trace from the build error identifies the line of code where the error occurs. If the error doesn't reference your project files, it's an issue with LEAN or another package.

Logic Errors

Logic errors occur when your algorithm behaves in an unexpected or unintended manner. These types of errors don't halt the program execution, so they are difficult to diagnose. An example code snippet that produces a logic error is the following:

var average = x + y / 2; // instead of (x + y) / 2
    
average = x + y / 2 # instead of (x + y) / 2

To resolve logic errors, trace the program carefully. You may use the Log and Debug methods or the built-in debugger.
