The debugger is a built-in tool to help you debug coding errors while backtesting. The debugger enables you to slow the code down, step through it line-by-line, and inspect the variables to understand the internal state of the program. The debugger accomplishes this by using breakpoints, watch variables, and stepping controls. Breakpoints are lines of code in your algorithm where execution will pause. Watch variables are the variables in your algorithm that you want to inspect during execution. The values of watch variables are formated in the IDE to improve readability. After you launch the debugger, you can step through your algorithm with the following controls:

  • Play button: Continue execution until the next breakpoint
  • Stop button: Exit the debugger
  • Arrow button: Step to the next line of code
