What are logging statements:
    - One-line statements that you can add to your algorithm to add to the log file, console, and (optionally) quit the program
    - They show up in the console and in the log files

Why use them:
     - To debug backtests
     - To debug live algorithms since you can't use the debugger.
     - They are are easy to add directly to your code files
          - You can add them directly under an `if` condition to signify an error has been seen

h3: Log
  - What is the Log statement?
      - A string that's added to the log file while the program continues execution
      - Because of data vendor limitations, price information cannot be recorded in logs

  - Why use the Log statement?
      - To debug backtests and live trading algorithms.

   - Table of daily/backtest log limits for each organization
       - If you hit the limit, contact support@quantconnect.com

    - If multiple identical messages are sent, they are filtered out. You can intentionally bypass this by adding a timestamp to your log messages.


h3: Debug
   - What is a Debug statement?
        - The same as Log statements but they are green in the console

   - Why use the Debug statement?
        - When you want to give more attention to an entry in the console
        - Contribute to log limits (?)

   - How(?):
       - capped at 200 characters long.
       - If multiple identical messages are sent within 1 second, we rate limit the messages to avoid crashing your browser


h3: Error
   - What is an Error statement?
        - The same as Log statements, but they are red in the console

   - Why use the Error statement?
        - When you want to give more attention to an entry in the console
        - Contribute to log limits (?)

   - How(?):
       - rate limited to avoid flooding your browser

h3: Quit
   - What is a Quit statement?
        - Quits the backtest (and may log some data to the log file)
        - Doesn't quit immediately, will proceed to end of method definition
        - Shows up as green in the console

   - Why use the Quit statement?
       - When you don't want the algorithm to continue execution
       - Contribute to log limits (?)

   - How(?):
      - Doesn't work in Initialize method
      - Doesn't work well in live mode (?)
      - Don't need to pass a string, can be empty
      - Example snippet
      - Doesn't quit immediately, will proceed to end of method definition
            - need to include a `return` statement directly after if you want to quit now
