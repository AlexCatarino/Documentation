The Research Environment is a Jupyter notebook-based environment where you can access our data through the QuantBook class instead of using the QCAlgorithm class in a backtest. The environment supports both Python and C#. To aid development, you can import code from the code files in your project into the Research Environment.

Before running backtests, we recommend testing your hypothesis in the Research Environment. It's easier to perform data analysis and produce plots in the Research Environment than in a backtest. In the Research Environment, you can use the QuantConnect API to import your backtest results for further analysis. You can also access Alpha Analytics without downloading anything to your local machine. Before backtesting or live trading machine learning models, you may find it beneficial to train them in the Research Environment, save them in the ObjectStore, and then load them from the ObjectStore into the backtesting and live trading environment before you run the backtest or deploy to live trading.

Note: This page is an introduction to the Research Environment for the Algorithm Lab. For more comprehensive information on using research notebooks, refer to the Research Environment documentation product.
