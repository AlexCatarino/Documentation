One of the drawbacks of using the Research Environment you may encounter is the need to rewrite code youâ€™ve already written in a file in the backtesting environment. Instead of rewriting the code, you can reuse the methods from the backtesting environment in the Research Environment to reduce development time. For example, say you have the following helpers.py file in your project:

def Add(a, b): 
   return a+b

To import the preceding method into your research notebook, use the import statement.

from helpers import Add

# reuse method from helpers.py
Add(3, 4)

If you adjust the file that you import, restart the Research Environment session to import the latest version of the file.
