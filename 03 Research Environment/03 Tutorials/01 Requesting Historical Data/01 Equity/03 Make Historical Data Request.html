There are three available ways to make historical data requests.

Bar count

The historical data will have the same length as defined by the bar count looking for data until the current algorithm time. It excludes the period the exchange was closed.

var history = qb.History(symbol, 10, Resolution.Daily);

history = qb.History(symbol, 10, Resolution.Daily)

Timespan

The first datapoint of the the historical data is defined by the timespan from the current algorithm time.

var history = qb.History(symbol, TimeSpan.FromDays(10), Resolution.Daily);

history = qb.History(symbol, timedelta(days=10), Resolution.Daily)

Defined period

The Historical data contains datapoints between the defined period.

var start = new DateTime(2021, 1, 1);
var end = new DateTime(2021, 2, 1);
var history = qb.History(symbol, start, end, Resolution.Daily);

start = datetime(2021, 1, 1)
end = datetime(2021, 2, 1)
history = qb.History(symbol, start, end, Resolution.Daily)

Please note that all of those ways require the definition of a resolution. If none is defined, the History method will assume the data subscription resolution.
