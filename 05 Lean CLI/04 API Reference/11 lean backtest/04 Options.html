The lean backtest command supports the following options:

Option Description
--output <path> Directory to store results in (defaults to <project>/backtests/<timestamp>).
--debug Enable a certain debugging method, see Debugging local backtests for more information.
--update Pull the selected LEAN engine version before running the backtest.
--version <value> The LEAN engine version to run (defaults to the latest installed version). This must be a valid tag of the quantconnect/lean Docker image.
--lean-config <path> The Lean configuration file that should be used (defaults to the nearest lean.json file).
--verbose Enable debug logging.
--help Display the help text of the lean backtest command and exit.
