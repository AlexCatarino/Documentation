Runs the LEAN Report Creator in a Docker container using the quantconnect/lean Docker image. The --backtest-data-source-file <path> is the only required option and should point to the JSON file containing backtest results. If --live-data-source-file <path> is also given the generated report will contain the combined data from both data sources.

The --strategy-name, --strategy-version, and --strategy-description options can be used to set the name, version, and description that are shown in the report. The name and version are shown in the top-right corner of each page, while the description is shown on the top of the first page. These fields are blank by default.

When the given backtest data source file is stored in a project directory or one of its subdirectories, the default name is the name of the project directory and the default description is the description in the project's config.json file.

By default, the latest installed version of the quantconnect/lean Docker image is used. If this image doesn't exist locally yet or if you specify the --update option it is pulled before running the backtest. You can specify a custom tag to use with the --version option.
