The lean report command supports the following options:

Option Description
--backtest-data-source-file <path> The path to the JSON file containing the backtest results (required).
--live-data-source-file <path> The path to the JSON file containing the live trading results.
--report-destination <path> The path where the generated report is stored as HTML (defaults to ./report.html).
--strategy-name <value> The name of the strategy, will appear at the top-right corner of each page.
--strategy-version <value> The version of the strategy, will appear next to the project name.
--strategy-description <value> The description of the strategy, will appear under the "Strategy Description" section on the first page of the report.
--overwrite Overwrite the given --report-destination if it already contains a file.
--update Pull the selected LEAN engine version before running the report creator.
--version <value> The version of the LEAN engine to run (defaults to the latest installed version). This version must be a valid tag of the quantconnect/lean Docker image.
--lean-config <path> The Lean configuration file that should be used (defaults to the nearest lean.json file).
--verbose Enable debug logging.
--help Display the help text of the lean report command and exit.
