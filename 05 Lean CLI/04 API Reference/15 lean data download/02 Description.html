Lets you purchase and download data from the QuantConnect Data Library. This command performs the following actions:

  1. You are asked to select the organization which should be billed. This step is skipped if you are only a member of a single organization.
  2. An interactive wizard is shown allowing you to configure exactly which data you want to download.
  3. You are asked to accept the CLI API Access and Data Agreement in your web browser.
  4. You are asked to confirm the purchase one last time.
  5. The CLI downloads the requested files while charging the organization selected in step 1.

The available data and pricing information can be found in the user guides. Data is priced on a per-file per-download basis, meaning you pay a certain number of QuantConnect Credits (QCC) for each file you download. The required QCC is deducted from your organization's QCC balance when a file is downloaded. If you force-quit the command before it downloaded all requested files, you are not charged for the files you didn't download.

When --product is given the command runs in non-interactive mode and step 1, 2, and 4 of the list above are skipped. Instead, the command reads the downloading configuration from the given command-line options. In this mode --product and --organization are required options, as well as all options specific to the selected product.

The following options are required for each product in non-interactive mode:

  • --product "CFD"
    • --ticker: must be set to the ticker of the data that you want to download.
    • --resolution: must be set to the resolution of the data that you want to download.
    • --start: must be set to the inclusive start date of the data that you want to download (only for tick, second, and minute resolutions).
    • --end: must be set to the inclusive end date of the data that you want to download (only for tick, second, and minute resolutions).
  • --product "Crypto"
    • --data-type: must be set to the type of data that you want to download, which can be one of Trade and Quote.
    • --market: must be set to the market of data that you want to download, which can be one of Bitfinex and GDAX.
    • --ticker: must be set to the ticker of the data that you want to download.
    • --resolution: must be set to the resolution of the data that you want to download.
    • --start: must be set to the inclusive start date of the data that you want to download (only for tick, second, and minute resolutions).
    • --end: must be set to the inclusive end date of the data that you want to download (only for tick, second, and minute resolutions).
  • --product "Equity"
    • --data-type: must be set to the type of data that you want to download, which can be one of Trade and Quote.
    • --ticker: must be set to the ticker of the data that you want to download.
    • --resolution: must be set to the resolution of the data that you want to download. Hourly and daily data is only available if --data-type is set to Trade.
    • --start: must be set to the inclusive start date of the data that you want to download (only for tick, second, and minute resolutions).
    • --end: must be set to the inclusive end date of the data that you want to download (only for tick, second, and minute resolutions).
  • --product "Equity Option"
    • --data-type: must be set to the type of data that you want to download, which can be one of Trade, Quote, and OpenInterest.
    • --option-style: must be set to the option style of the data that you want to download, which can be one of American and European.
    • --ticker: must be set to the ticker of the data that you want to download.
    • --start: must be set to the inclusive start date of the data that you want to download.
    • --end: must be set to the inclusive end date of the data that you want to download.
  • --product "Forex"
    • --ticker: must be set to the ticker of the data that you want to download.
    • --resolution: must be set to the resolution of the data that you want to download.
    • --start: must be set to the inclusive start date of the data that you want to download (only for tick, second, and minute resolutions).
    • --end: must be set to the inclusive end date of the data that you want to download (only for tick, second, and minute resolutions).
  • --product "Future"
    • --market: must be set to the market of the data that you want to download, which can be one of CBOE, CBOT, CME, COMEX, HKFE, ICE, NYMEX, and SGX.
    • --ticker: must be set to the ticker of the data that you want to download the margins of.
  • --product "CBOE Volatility Index Pricing"
    • --ticker: must be set to the ticker of the index.
  • --product "Federal Reserve Economics Data"
    • --ticker: must be set to the name of the variable passed to AddData(Fred, <variable>) (example: "Fred.CBOE.VIXOnGoogle") or to the series id used by FRED (example: "VXGOGCLS").
  • --product "SEC Filings"
    • --report-type: must be set to the type of the reports to download, which can be one of 10K, 10Q, and 8K.
    • --ticker: must be set to the ticker of the company to download the reports for.
    • --start: must be set to the inclusive start date of the data.
    • --end: must be set to the inclusive end date of the data.
  • --product "US Treasury Yield Curve Rates"
    • No required options.
  • --product "US Energy Information Administration Data"
    • --ticker: must be set to the name of the variable passed to AddData(USEnergy, <variable>) (example: "USEnergy.Petroleum.UnitedStates.WeeklyImportsOfCrudeOil") or to the series id used by the US Energy Information Administration (example: "PET.WCRIMUS2.W").

Example non-interactive usage: 

$ lean data download \
    --product "Crypto" \
    --organization "My Organization" \
    --data-type "Trade" \
    --market "GDAX" \
    --ticker "BTCUSD" \
    --resolution "Daily"

In case the local data already exists a warning is logged and you are given the choice of whether you want to enable overwriting existing data or not. Use the --overwrite flag to override this behavior and enable overwriting existing data in all such cases.
