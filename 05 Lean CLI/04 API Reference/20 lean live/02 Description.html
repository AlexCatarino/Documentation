Starts local live trading in a Docker container using the quantconnect/lean Docker image. The logs of the algorithm are shown in real-time and the full results are stored in the <project>/live/<timestamp> directory. You can use the --output option to change the output directory.

The given <project> argument must be either a project directory or a file containing the algorithm to run a backtest for. If it is a project directory the CLI looks for a main.py or Main.cs file, assuming the first file it finds to contain the algorithm to run.

The given environment must be one of the live environments stored in your Lean configuration file. This means the environment must have the live-mode property set to true. For most brokerages, you also need to configure some additional properties in the Lean configuration file, like your brokerage credentials. The required environment and properties for each brokerage are listed on the Local live trading page.

The Docker image that is used contains the same libraries as the ones available on QuantConnect. If the selected project is a C# project the project is first compiled in a separate container, but using the same image.

By default, the latest installed version of the quantconnect/lean Docker image is used. If this image doesn't exist locally yet or if you specify the --update option it is pulled before starting the local live trading. You can specify a custom tag to use with the --version option.
