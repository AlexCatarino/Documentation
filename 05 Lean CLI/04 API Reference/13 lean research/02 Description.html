Runs a local Jupyter Lab environment in a Docker container using the quantconnect/research Docker image. This mounts the entire project directory in the Docker container and exposes the Jupyter Lab instance on a local port. After the Jupyter Lab instance has started the browser is automatically opened.

By default Jupyter Lab is exposed on port 8888. If you want to use a custom port you can use the --port option, which may be useful if you want to run two Jupyter Lab instances side-by-side.

If you have previously logged in using lean login the CLI automatically makes your credentials available in the Jupyter Lab instance. This means the api variable is automatically assigned an instance of Api in your research notebooks which you can use to make authenticated requests to the QuantConnect API.

By default the latest installed version of the quantconnect/research Docker image is used. If this image doesn't exist locally yet or if you specify the --update option it is pulled before starting the research environment. You can specify a custom tag to use with the --version option.
