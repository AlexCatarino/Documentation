The lean data generate command supports the following options:

Option Description
--start <yyyyMMdd> The inclusive start date for the data to generate in yyyyMMdd format.
--end <yyyyMMdd> The inclusive end date for the data to generate in yyyyMMdd format (defaults to today).
--symbol-count <value> The number of symbols to generate data for.
--security-type <value> The security type to generate data for (defaults to Equity). Must be one of Equity, Forex, Cfd, Future, Crypto or Option.
--resolution <value> The resolution of the generated data (defaults to Minute). Must be one of Tick, Second, Minute, Hour or Daily. See the description for the supported combinations of security types and resolutions.
--data-density <value> The density of the generated data (defaults to Dense). Must be one of Dense (at least one data point per resolution step), Sparse (at least one data point per 5 resolution steps), or VerySparse (at least one data point per 50 resolution steps).
--update Pull the selected LEAN engine version before running the backtest.
--version <value> The LEAN engine version to run the random data generator of (defaults to the latest installed version). This version must be a valid tag of the quantconnect/lean Docker image.
--lean-config <path> The Lean configuration file that should be used (defaults to the nearest lean.json file).
--verbose Enable debug logging.
--help Display the help text of the lean data generate command and exit.
