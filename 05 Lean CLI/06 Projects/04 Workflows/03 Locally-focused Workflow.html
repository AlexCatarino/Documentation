A locally-focused workflow (local development, local execution) with the CLI might look like this:

  1. Open a terminal in the directory you ran lean init in.
  2. Run lean create-project "<projectName>" to create a new project with some basic code to get you started.
  3. Work on your strategy in ./<projectName>.
  4. Run lean research "<projectName>" to start a Jupyter Lab session to perform research.
  5. Run lean backtest "<projectName>" to run a backtest whenever there's something to test. This command runs your strategy in a Docker container containing the same packages as the ones used on QuantConnect.com, but with your own data.

With this workflow, you are not limited to the computing power that's available in QuantConnect's infrastructure, because everything runs locally. On the other hand, this also means you must have all your required data available locally. To download some of QuantConnect's data for local usage, see Downloading QuantConnect Data.
