Run lean backtest "Project Name" to run a local backtest for the given project. This will automatically run a backtest for the specified project with the LEAN engine in the latest downloaded version of the quantconnect/lean Docker image. By default the results of the backtests and its logging is <project>/backtests/<timestamp> directory. The --output <directory> option can be used to write output to a different directory.

If the specified project is a C# project, the CLI will first compile it. This happens in a separate Docker container, but using the same image as the one that is used to run the backtest. This means your C# code has access to all the libraries available on QuantConnect (apart from the classes in System.* and QuantConnect.*).

You can use the --version option if you want to run a specific version of the quantconnect/lean image. The value of the option should be the tag that should be used, so lean backtest "Project Name" --version 10992 will run the 10992 tag.
