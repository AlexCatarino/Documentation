Run lean backtest "Project Name" to run a local backtest for the given project. This command automatically runs a backtest for the specified project with the LEAN engine in the quantconnect/lean Docker image. This Docker image contains all the libraries available on QuantConnect, meaning your algorithm also has access to those libraries. If the specified project is a C# project it is compiled first.

By default, the results of the backtests and their logging are stored in the <project>/backtests/<timestamp> directory. The --output <directory> option can be used to write output to a different directory.

You can use the --version option to run a specific version of the quantconnect/lean image. The value of the option is the tag that is used, so lean backtest "Project Name" --version 10992 runs the 10992 tag.
