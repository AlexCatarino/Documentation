Run lean report --backtest-data-source-file <backtest file> to create a report, where <backtest file> should point to the output file of a locally ran backtest. This file is usually the largest JSON file in the output directory of a backtest. If you also want to include live trading results in the report you can enable that by providing --live-data-source-file <path>, where <path> should point to the JSON file containing the live trading results. By default, the generated report is saved to ./report.html, although you can change the output location by specifying a custom path with --report-destination <path>.

When generating a report you can also configure the following details, which are blank by default:

  • Strategy name: This name is displayed in the top-right corner of each page, and can be configured using --strategy-name <value>.
  • Strategy version: This version is displayed next to the strategy name and can be configured using --strategy-version <value>.
  • Strategy description: This description is displayed underneath the "Strategy Description" header on the first page and can be configured using --strategy-description.

When the given backtest data source file is stored a project directory (like <project>/backtests/<timestamp>), the default name is the name of the project directory and the default description is the description stored in the project's configuration.
