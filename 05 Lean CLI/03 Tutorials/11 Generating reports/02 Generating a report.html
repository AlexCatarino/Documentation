Follow these steps to generate a report of a backtest:

  1. Open a terminal in the directory you ran lean init in.
  2. Find the path to the backtest results to display in the report. Usually, this is the largest JSON file in the output directory of the backtest, which defaults to <project>/backtests/<timestamp>.
  3. Run lean report --backtest-data-source-file <backtest file> to generate a report. 
    $ lean report --backtest-data-source-file "My Project/backtests/2021-03-21_04-09-10/main.json"
20210322 20:03:48.718 TRACE:: QuantConnect.Report.Main(): Parsing source files...backtest-data-source-file.json,
20210322 20:03:51.602 TRACE:: QuantConnect.Report.Main(): Instantiating report...
Successfully generated report to './report.html'
    By default, the generated report is saved to ./report.html, although you can change this by providing a custom path with the --report-destination <path> option.
  4. Open the generated report in the browser and inspect its results.

Optionally you can also configure the following details, which are blank by default:

  • Strategy name: This name is displayed in the top-right corner of each page, and can be configured using --strategy-name <value>.
  • Strategy version: This version is displayed next to the strategy name and can be configured using --strategy-version <value>.
  • Strategy description: This description is displayed underneath the "Strategy Description" header on the first page and can be configured using --strategy-description.
  • Live results: These results are displayed over the backtest results, and can be configured using --live-data-source-file <path>. The provided path must point to a JSON file containing live results.

When the given backtest data source file is stored a project directory (like <project>/backtests/<timestamp>), the default name is the name of the project directory and the default description is the description stored in the project's configuration.
