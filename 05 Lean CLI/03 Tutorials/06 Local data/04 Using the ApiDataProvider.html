An alternative to manually downloading all required data before running the backtest is to use the ApiDataProvider. This data provider automatically downloads the data files that you added to your account when your backtest needs it. After it downloads a data file, it stores it in your local data directory so that in future runs it'll not have to download it again. Follow these steps to set up the ApiDataProvider:

  1. Log in to the CLI if you haven't done so already.
  2. Open the lean.json file that was generated when you ran lean init.
  3. Update the data-provider to QuantConnect.Lean.Engine.DataFeeds.ApiDataProvider in the lean.json file.
  4. Save the lean.json file. Your local LEAN instance now automatically uses the ApiDataProvider when you run it through the CLI.
