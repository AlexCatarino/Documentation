An alternative to manually downloading all required data before running the backtest is to use the ApiDataProvider. This is data provider in LEAN which automatically downloads the data files that you added to your account when your backtest needs it. After it downloads a data file, it stores it in your local data directory so that in future runs it'll not have to download it again.

You need to be logged in with lean login before you can use the ApiDataProvider. When you are logged in, open the lean.json file that was generated when you ran lean init. In the lean.json file, set the value of the data-provider to QuantConnect.Lean.Engine.DataFeeds.ApiDataProvider and save the file. After updating the lean.json file your local LEAN instance will automatically use the ApiDataProvider when you run it through the CLI.
