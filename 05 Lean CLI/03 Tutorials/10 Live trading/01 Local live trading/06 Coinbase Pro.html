Make sure the following environment exists in your lean.json file before you start local live trading on Coinbase Pro: 

{
    "environments": {
        "live-gdax": {
            "live-mode": true,

            "live-mode-brokerage": "GDAXBrokerage",
            "data-queue-handler": "GDAXDataQueueHandler",
            "setup-handler": "QuantConnect.Lean.Engine.Setup.BrokerageSetupHandler",
            "result-handler": "QuantConnect.Lean.Engine.Results.LiveTradingResultHandler",
            "data-feed-handler": "QuantConnect.Lean.Engine.DataFeeds.LiveTradingDataFeed",
            "real-time-handler": "QuantConnect.Lean.Engine.RealTime.LiveTradingRealTimeHandler",
            "transaction-handler": "QuantConnect.Lean.Engine.TransactionHandlers.BrokerageTransactionHandler",
            "history-provider": "BrokerageHistoryProvider"
        }
    }
}

Make sure the following properties are configured above the environments object in your lean.json file before you start local live trading on Coinbase Pro:

  • gdax-api-secret: This property must contain your API secret. You can create a new API key on pro.coinbase.com/profile/api.
  • gdax-api-key: This property must contain the API key belonging to the given secret.
  • gdax-passphrase: This property must contain the passphrase belonging to the given API key and secret.

After setting up the required configuration you can continue with step 4 of the tutorial in the introduction and use live-gdax as environment name in the steps that follow.
