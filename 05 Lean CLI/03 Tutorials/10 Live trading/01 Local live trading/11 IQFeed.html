Instead of using the data feed from your brokerage, you can also use IQFeed if you're on Windows. Using IQFeed requires you to have an IQFeed developer account and you need to have the IQFeed client installed locally. Follow these steps to set up IQFeed as your data feed:

  1. Perform steps 1 through 4 of the tutorial in the introduction.
  2. In the environment settings configured in step 4 of the tutorial, set data-queue-handler to QuantConnect.ToolBox.IQFeed.IQFeedDataQueueHandler and history-provider to QuantConnect.ToolBox.IQFeed.IQFeedDataQueueHandler. 
    {
    "environments": {
        "environment-name": {
            "live-mode": true,

            "data-queue-handler": "QuantConnect.ToolBox.IQFeed.IQFeedDataQueueHandler",
            "history-provider": "QuantConnect.ToolBox.IQFeed.IQFeedDataQueueHandler"
        }
    }
}
  3. Make sure the following properties are configured above the environments object in your lean.json file:
    • iqfeed-iqconnect: This property must contain the path to the IQConnect binary. The default path is C:/Program Files (x86)/DTN/IQFeed/iqconnect.exe if you used the default settings when installing the IQFeed client.
    • iqfeed-username: This optional property may contain your IQFeed username. IQConnect prompts you for it before live trading starts if it is not set.
    • iqfeed-password: This optional property may contain your IQFeed password. IQConnect prompts you for it before live trading starts if it is not set.
    • iqfeed-productName: This property must contain the product id of your IQFeed developer account.
    • iqfeed-version: This property must contain the version of the product given by iqfeed-productName.
  4. Continue with step 5 of the tutorial in the introduction.
