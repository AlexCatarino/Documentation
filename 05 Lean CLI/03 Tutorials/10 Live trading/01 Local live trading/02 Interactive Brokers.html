To start local live trading on Interactive Brokers, make sure the following environment exists: 

{
    "environments": {
        "live-interactive": {
            "live-mode": true,

            "live-mode-brokerage": "InteractiveBrokersBrokerage",
            "setup-handler": "QuantConnect.Lean.Engine.Setup.BrokerageSetupHandler",
            "result-handler": "QuantConnect.Lean.Engine.Results.LiveTradingResultHandler",
            "data-feed-handler": "QuantConnect.Lean.Engine.DataFeeds.LiveTradingDataFeed",
            "data-queue-handler": "QuantConnect.Brokerages.InteractiveBrokers.InteractiveBrokersBrokerage",
            "real-time-handler": "QuantConnect.Lean.Engine.RealTime.LiveTradingRealTimeHandler",
            "transaction-handler": "QuantConnect.Lean.Engine.TransactionHandlers.BrokerageTransactionHandler",
            "history-provider": "BrokerageHistoryProvider"
        }
    }
}

The following properties should be configured before you start local live trading on Interactive Brokers:

  • ib-account: This property must contain the name of the account (example: DU1234567).
  • ib-user-name: This property must contain the username of the given account (example: trader123).
  • ib-password: This property must contain the password of the given account.
  • ib-agent-description: This property describes the type of trader that you are. It must be one of the following values:
    • Individual or I for an individual. If you're not aware what the below means, you probably need this value.
    • A for an agency.
    • W for an agent or other member.
    • J for an individual PTIA.
    • U for an agency PTIA.
    • M for an agent or other member PTIA.
    • K for an individual PT.
    • Y for an agency PT.
    • N for an agent or other member PT.
  • ib-trading-mode: This property must be paper if the specified account is a paper account, or live if the specified account is a live account.
  • ib-enable-delayed-streaming-data: This property configures the behavior when your algorithm attempts to subscribe to market data which you don't have a market data subscription on Interactive Brokers for. When set to true, your algorithm continues running using delayed market data. When set to false, live trading will stop and LEAN will shut down.

After setting up all the above, you can start local live trading on Interactive Brokers by running lean live <project name> live-interactive.
