The CLI contains a lot of commands to make working on LEAN algorithms easier and more productive. Below we list some of the most common tasks, see the tutorials in the sidebar and the command reference for a complete overview of the supported features.

Authenticating with the cloud

Before using the CLI to perform tasks in the cloud it is required to login with your QuantConnect API credentials. Run lean login to open an interactive wizard which will ask you for your user id and API token. Both of these credentials can be requested on quantconnect.com/account, after which they'll be sent to your email address.

Pulling projects from the cloud

Run lean cloud pull to pull your QuantConnect projects to your local drive. This command pulls all your cloud projects to your local drive while preserving your QuantConnect directory structure. If you have a lot of projects and only want to work locally on a few of them you can run this command with the --project "Project Name" option, which will make the command pull a single project instead.

Downloading data from the cloud

This feature is not implemented yet.

Running a local backtest

Run lean backtest "Project Name" to run a local backtest for the specified project. This command runs a backtest in a Docker container containing the same packages as the ones used on QuantConnect.com, but with your own data.

Pushing local changes to the cloud

Run lean cloud push to push local changes to the QuantConnect. This command pushes all your local projects to the cloud and creates new cloud projects when necessary. If you only want to push a single project you can run this command with the --project "Project Name" command.

Running a cloud backtest

Run lean cloud backtest "Project Name" to run a cloud backtest for the specified project. By default a summary of the results and a link to the full results is shown in the terminal. Running this command with the --open flag will automatically open the full results in the browser once the backtest is finished. Additionally you can run this command with the --push flag to push all local changes to the project to the cloud before running the backtest.
