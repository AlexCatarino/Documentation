The following errors are commonly seen when using the CLI:

Error text Possible cause and fix
  • No such command '<name>'
  • No such option: <option>
 The command you tried to run does not exist or it doesn't support the option you provided. If the documentation says it is available you are probably using an outdated version of the CLI. Run pip install --upgrade lean to update to the latest version.
Invalid credentials, please log in using `lean login` You are trying to use a command which communicates with the QuantConnect API and you haven't authenticated yourself yet. Run lean login to log in with your API credentials.
Please make sure Docker is installed and running You are trying to use a command which needs to run the LEAN engine locally. This is done in a Docker container, which requires Docker to be installed and running. Make sure Docker is running if you installed it already, or visit installing Docker if you haven't.
This command should be executed in a Lean CLI project, run `lean init` in an empty directory to create one or specify the Lean configuration file to use with --lean-config The command you are trying to run requires a Lean configuration file. The CLI will automatically try to find this by recursively looking in the current directory and all of the parent directories for a lean.json file. If no such file can be found, this error will be shown. It can be fixed by running the command in a directory you have previously executed lean init in (which generates the lean.json file), or by specifying the path to the lean.json file to use with the --lean-config option.
