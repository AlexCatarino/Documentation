After running lean init the following structure is created: 

.
├── .idea/
│   ├── .idea.LeanCLI.dir/
│   │   └── .idea/
│   │       └── workspace.xml
│   └── workspace.xml
├── .vscode/
│   ├── launch.json
│   └── settings.json
├── data/
│   ├── alternative/
│   ├── cfd/
│   ├── crypto/
│   ├── equity/
│   ├── forex/
│   ├── futureoption/
│   ├── market-hours/
│   ├── option/
│   ├── symbol-properties/
│   └── readme.md
├── LeanCLI.csproj
└── lean.json

These files contain the following content:

  • .idea/.idea.LeanCLI.dir/.idea/workspace.xml: this file contains debug configuration to make debugging C# with Rider easier.
  • .idea/workspace.xml: this file contains debug configuration to make debugging Python with PyCharm easier.
  • .vscode/launch.json: this file contains debug configuration to make debugging Python and C# with VS Code easier.
  • .vscode/settings.json: this file contains VS Code configuration so that VS Code's Python extension can provide accurate autocompletion.
  • LeanCLI.csproj: this file contains configuration which Visual Studio, Rider and VS Code can read to provide accurate C# autocompletion.
  • data/: this directory contains the local data that is used when running the LEAN engine locally. This directory is filled with sample data from the QuantConnect/Lean repository.
  • lean.json: this file contains the configuration that is used when running the LEAN engine locally.

It is okay to delete files you don't need. If you deleted an unnecessary file in the past and realize at a later time that you do need it, simply run lean init in an empty directory again and copy over the files you need.
