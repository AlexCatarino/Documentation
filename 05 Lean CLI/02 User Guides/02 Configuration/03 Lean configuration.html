The Lean configuration contains the settings to use when running the LEAN engine locally. This configuration is created in the lean.json file when you run lean init in an empty directory. The configuration is stored as JSON, with support for both single-line and multiline comments.

The Lean configuration file is based on the Launcher/config.json file from the Lean repository. When you run lean init the latest version of this file is downloaded and stored on your local drive. Before the file is stored some properties are automatically removed because the CLI can set them automatically based on the command that you run or because the CLI saves the configuration somewhere else. This affects the following properties:

  • environment: this property is automatically set to backtesting when running a backtest or an optimization, or to the environment you provide to the CLI when starting local live trading.
  • composer-dll-directory: this property is automatically set to the right directory when running lean research.
  • algorithm-type-name, algorithm-language and algorithm-location: these properties are automatically set based on the project name you give to the CLI when running a backtest/optimization or when starting local live trading.
  • debugging and debugging-method: these properties are automatically set to the appropriate values when debugging a local backtest.
  • job-user-id and api-access-token: these properties are automatically set to the credentials provided via lean login.
  • parameters: this property is stored in the project configuration and automatically added to the Lean configuration when running the LEAN engine locally.
