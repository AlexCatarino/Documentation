When you run lean init in an empty directory, the CLI will download the latest data directory from the LEAN repository. This directory contains a standard directory structure which the LEAN engine can read from. Because LEAN strives to use an open, human readible format all data is stored in flat files (formatted as CSV or JSON), and compression is done using zip. Once downloaded, the data directory tree will look like this:

├── alternative/
│   ├── estimize/
│   ├── sec/
│   └── trading-economics/
├── cfd/
│   ├── oanda/
│   └── readme.md
├── crypto/
│   ├── gdax/
│   └── readme.md
├── equity/
│   ├── usa/
│   └── readme.md
├── forex/
│   ├── fxcm/
│   ├── oanda/
│   └── readme.md
├── future/
│   ├── cboe/
│   ├── cbot/
│   ├── cme/
│   ├── comex/
│   ├── hkfe/
│   ├── ice/
│   ├── nymex/

├── symbol-properties/
│   └── symbol-properties-database.csv
└── readme.md

By default the data directory will contain a small amount of sample data for all asset types to demonstrate how files should be formatted. Additionally, the data directory itself and most of its subdirectories contain readme.md files containing additional documentation on the format of the data files of each asset type.
