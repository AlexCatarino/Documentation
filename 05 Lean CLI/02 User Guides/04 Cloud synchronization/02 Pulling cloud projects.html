When you run lean cloud pull to download cloud projects to your local drive, the CLI attempts to download each of them to the local directory matching the cloud project's path. If one of your cloud project is named "My Strategy" and stored in the "Strategies" directory, the CLI attempts to download it to ./Strategies/My Strategy. Before writing the cloud project's files to the local directory, the CLI performs a set of checks to make sure it doesn't accidentally overwrite the wrong files. To perform these checks the following information is gathered:

  • Whether the local directory for the cloud project exists or not.
  • Whether the local directory contains a config.json file with a cloud-id matching the cloud project's id.

The CLI only proceeds with downloading the cloud project if and only if the local directory does not exist yet or the cloud-id in the config.json file matches the cloud project's id. If these criteria are not met, a warning is shown to let you know exactly what went wrong and the cloud project is not downloaded.

When the required criteria are met, the CLI downloads all cloud project's files and parameters to the local directory, potentially overwriting existing files and parameters. Local files which do not exist in the cloud are not deleted. After successfully pulling the cloud project's files and parameters, the cloud project's id is saved in the local directory's config.json file. This ensures the CLI knows which cloud project is linked to the local project the next time you try to pull/push the project.
