By default, LEAN adds contracts to the OptionChain that pass the filter criteria at every time step in your algorithm. If a contract has been in the universe for a duration that matches the MinimumTimeInUniverse setting and it no longer passes the filter criteria, LEAN removes it from the chain.

The default behavior leads to a large number of open data subscriptions. In live trading, it can be inconvenient because of data subscription limits imposed by data feed providers. To reduce the number of data subscriptions, change the value of MinimumTimeInUniverse.

UniverseSettings.MinimumTimeInUniverse = TimeSpan.FromMinutes(15);
UniverseSettings.MinimumTimeInUniverse = TimeSpan.Zero;
self.UniverseSettings.MinimumTimeInUniverse = timedelta(minutes=15)
self.UniverseSettings.MinimumTimeInUniverse = timedelta(0)

If you set the value to TimeSpan.Zerotimedelta(0) and LEAN readd a subscription removed at the previous filter evaluation, your algorithm will not receive the data for the contract when it would be immediately available with a higher MinimumTimeInUniverse.
