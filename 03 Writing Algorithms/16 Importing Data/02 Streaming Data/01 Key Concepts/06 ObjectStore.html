It is optional to first transfer remote files in other locations (e.g. dropbox) to ObjectStore. It can increase the execution speed of your backtests since it don't have to download the data at each run.

Manually, you can do so by uploading the files into the ObjectStore via the Organization page.

Alternatively, you can also do so by a bulk download first. Then, re-uploading into the ObjectStore. It can be done using the research environment easily.

// Case 1: CSV file
var csvStringFile = qb.Download("<csvFileURL>");
var csvSaveSuccessful = qb.ObjectStore.Save($"{qb.ProjectId}/{"<YourCSVKey>"}", csvStringFile);

// Case 2: JSON file
var jsonFile = qb.Download("<jsonFileURL>");
var jsonSaveSuccessful = qb.ObjectStore.SaveJson<Dictionary<string, decimal>>($"{qb.ProjectId}/{"<YourJSONKey>"}", jsonFile);
# Case 1: CSV file
csv_string_file = qb.download("<csv_file_url>")
csv_save_successful = qb.object_store.save(f'{qb.project_id}/{"<your_csv_key>"}', csv_string_file)

# Case 2: JSON file
json_file = qb.download("<json_file_url>")
json_save_successful = qb.object_store.save(f'{qb.project_id}/{"<your_json_key>"}', json_file)

To load the data in backtesting, set the custom data source to read from the saved file in ObjectStore and parse the data just like parsing from other data sources.

public class MyCustomDataType : BaseData
{
    public override SubscriptionDataSource GetSource(
        SubscriptionDataConfig config,
        DateTime date,
        bool isLiveMode)
    {
        return new SubscriptionDataSource("<YourCSVKey>", SubscriptionTransportMedium.ObjectStore, FileFormat.Csv);
        // return new SubscriptionDataSource("<YourJSONKey>", SubscriptionTransportMedium.ObjectStore, FileFormat.UnfoldingCollection);
    }
}
class MyCustomDataType(PythonData):

            config: SubscriptionDataConfig,
            date: datetime,
