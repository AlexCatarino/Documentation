The following examples demonstrate some common practices for requesting individual Future contract data.

Example 1: Extended Hours Continuous Future Contract

The following example demonstrates how to request data subscription of E-Mini S&P 500 Future contract with continuous contract mapped by open interest using the dataMappingModedata_mapping_mode argument. To also subscribe to extended market hours data, set the extendedMarketHoursextended_market_hours argument to trueTrue.

public FutureExampleAlgorithm : QCAlgorithm
{
    private Symbol _future;

    public override void Initialize()
    {
        // Map the continuous Future contract by highest open interest allows better liquidity.
        // Usually it will be the front month contract, so it has lower interst rate and credit risk.
        // Extended market hours trading allows flexibility and hedging during equity market close.
        _future = AddFuture(Futures.Indices.SP500EMini,
            extendedMarketHours: true,
            dataMappingMode: DataMappingMode.OpenInterest).Symbol;
    }
}
class FutureExampleAlgorithm(QCAlgorithm):
    def initialize(self):
        # Map the continuous Future contract by highest open interest allows better liquidity.
        # Usually it will be the front month contract, so it has lower interst rate and credit risk.
        # Extended market hours trading allows flexibility and hedging during equity market close.
        self._future = self.add_future(Futures.Indices.SP_500_E_MINI,
            extended_market_hours=True,
            data_mapping_mode=DataMappingMode.OPEN_INTEREST).symbol
