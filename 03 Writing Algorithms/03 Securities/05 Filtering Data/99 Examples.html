Example 1: Filter Price Outliners Using SMA

When analyzing high-frequency price data, it's important to filter out potential outliers and anomalies that may skew the analysis. One effective method is to use a 1-month SMA to identify price ticks that deviate significantly from the short-term trend. By comparing each price tick to the 1-month SMA, you can flag any data points that fall outside a specified threshold, which we used 3 standard deviations in this example. This helps remove suspicious or erroneous price information, ensuring a cleaner dataset for further technical analysis and trading strategies.

public class SampleAlgorithm : QCAlgorithm
{
    public override void Initialize()
    {
        dynamic equity = AddEquity("AAPL", Resolution.Daily);
        // Set a 22-step for 1 month daily price SMA
        equity.Sma = SMA(equity.Symbol, 22, Resolution.Daily);
        // Set 22-step rolling window of the SMA indicator to save last month data to calculate standard deviation
        equity.Sma.Window.Size = 22;
        // Warm up the indicator
        WarmUpIndicator((Symbol)equity.Symbol, (SimpleMovingAverage)equity.Sma, Resolution.Daily);
        // Set the data filter
        equity.SetDataFilter(new CustomDataFilter());
    }

    private class CustomDataFilter : ISecurityDataFilter
    {
        public bool Filter(Security vehicle, BaseData data)
        {
            // Cast into dynamic for custom property SMA
            var security = vehicle as dynamic;
            // Get the average and standard deviation of the SMA data points
            var sma = security.Sma.Current.Value;
            var window = (RollingWindow<IndicatorDataPoint>)security.Sma.Window;
            var std = Convert.ToDecimal(Math.Sqrt(window.Average(point => Math.Pow((double)(point.Value - sma), 2))));
            
            // Check if the data point is suspicious
            if (data.Value.IsSuspicious(sma, sd))
            {
                // If so, we reject the data
                return false;
            }
            // Otherwise, we accept it
            return true;
        }

        private IsSuspicious(this decimal value, decimal sma, decimal sd)
        {
            // Define suspicious data point to be out of 3SD of the SMA
            return value > sma + 3m * std || value < sma - 3m * std
        }
    }
}

class CustomDataFilter(SecurityDataFilter):
    def filter(self, vehicle: Security, data: BaseData) -> bool:
        # Get the average and standard deviation of the SMA data points
        sma = vehicle.sma.current.value
        std = np.std([x.value for x in vehicle.sma.window])
        
        # Check if the data point is suspicious
        if is_suspicious(data.values, sma, std):
            # If so, we reject the data
            return False
        # Otherwise, we accept it
        return True
    
    def is_suspicious(self, value: float, sma: float, sd: float):
        # Define suspicious data point to be out of 3SD of the SMA
        return value > sma + 3 * std or value < sma - 3 * std

Example 2: Filter Major Exchanges

This examples shows how to only keep the data feed from BATS exchange while filtering the other major exchanges.

public override void Initialize()
{
    var security = AddEquity("SPY", Resolution.Tick);
    security.SetDataFilter(new CustomDataFilter());
}

private class CustomDataFilter : ISecurityDataFilter
{
    public bool Filter(Security vehicle, BaseData data)
    {
        // Use tick data to obtain the exchange of the feed
        var tick = data as Tick;
        
        // Check if the feed is from BATS
        if (tick != null && tick.Exchange == Exchange.BATS)
        {
            // If so, we accept the data
            return true;
        }
        // Otherwise, we reject it
        return false;
    }
}
def initialize(self):
    security = self.add_equity("SPY", Resolution.TICK)
    security.set_data_filter(CustomDataFilter())

class CustomDataFilter(SecurityDataFilter):
    def filter(self, vehicle: Security, data: BaseData) -> bool:
        # Use tick data to obtain the exchange of the feed
        tick = Tick(data)
        
        # Check if the feed is from BATS
        if tick and tick.exchange == Exchange.BATS:
            # If so, we accept the data
            return True
        # Otherwise, we reject it
        return False

When trading less liquid financial instruments, it can be advantageous to focus on the BATS exchange, as its quote data may not fully reflect the fair market value. Due to the lower trading volume and visibility of BATS, the quoted prices there may lag behind the true underlying value of illiquid assets. A carefully designed algorithm can analyze the BATS feed to identify situations where the quoted price appears to be undervalued compared to the asset's intrinsic worth. By executing trades to capture this disconnect, rather than arbitraging between exchanges, you may be able to profit from the market inefficiencies present in the less liquid instrument.
