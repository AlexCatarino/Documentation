The OnData method is the primary event handler for receiving financial data events to your algorithm. It is triggered sequentially at the point in time the data is available; in backtesting and live. 

public override void OnData(Slice data)
{
}

def OnData(self, data: Slice):
    pass

All data for a given moment of time is grouped in a single event, including custom data types. This data is passed with the Slice object.

If your algorithm code takes a long time to process, events will be queued up and the next event will be triggered again immediately. In live trading it is important to ensure your event handlers process quickly. Perform intensive computation before the market opens with a Scheduled Event, or a Train event.
