The following examples are typical filter functions you may want.

Example 1: Take 500 stocks that are worth more than $10 and have more than $10M daily trading volume

The most common use case is to select a lot of liquid stocks. With a fundamental universe filter, this is simple and fast. The following example selects the top most liquid 500 stocks over $10 per share. 

public class LiquidNonPennyStocksUniverseAlgorithm : QCAlgorithm
{
    public override void Initialize()
    {
        // Monthly updating the universe is sufficient, since the top 500 will not alter a lot in short time, such that we allow longer trades.
        UniverseSettings.Schedule.On(DateRules.MonthStart());
        // Add universe with custom selection rules for filtering.
        AddUniverse(FundamentalFilterFunction);
    }
    
    private IEnumerable<Symbol> FundamentalFilterFunction(IEnumerable<Fundamental> fundamental) 
    {
        // Filter for the most liquid stocks to trade with lower slippage and better fills.
        // Set threshold on non-penny stocks to reduce excessive volatility from penny stocks.
        return (from f in fundamental
            where f.Price > 10 && f.DollarVolume > 10000000
            orderby f.DollarVolume descending
            select f.Symbol).Take(500);
    }
}
class LiquidNonPennyStocksUniverseAlgorithm(QCAlgorithm):

    def initialize -> None:
        # Monthly updating the universe is sufficient, since the top 500 will not alter a lot in short time, such that we allow longer trades.
        self.universe_settings.schedule.on(self.date_rules.month_start())
        # Add universe with custom selection rules for filtering.
        self.add_universe(self._fundamental_filter_function)

    def _fundamental_filter_function(self, fundamental: List[Fundamental]) -> List[Symbol]:
        # Set threshold on non-penny stocks to reduce excessive volatility from penny stocks.
        filtered = [f for f in fundamental if f.price > 10 and f.dollar_volume > 10000000]
        # Filter for the most liquid stocks to trade with lower slippage and better fills.
        liquid_stocks = sorted(filtered, key=lambda f: f.dollar_volume, reverse=True)[:500]
        return liquid_stocks

Example 2: Take 10 stocks above their 200-Day EMA and have more than $1B daily trading volume

Another common request is to filter the universe by a technical indicator, such as only picking stocks above their 200-day EMA. The Fundamental object has adjusted price and volume information, so you can do any price-related analysis. 

public class UpTrendLiquidUniverseAlgorithm : QCAlgorithm
{    
    // A concurrent dictionary to save the EMA data asynchonously.
    private ConcurrentDictionary<Symbol, SelectionData> _state_data = new ConcurrentDictionary<Symbol, SelectionData>();

    public override void Initialize()
    {
        // Add universe with custom selection rules for filtering.
        AddUniverse(FundamentalFilterFunction);
    }
    
    private IEnumerable<Symbol> FundamentalFilterFunction(IEnumerable<Fundamental> fundamental)
    {
        // To create and update EMA indicators for each stocks for estimating their trend. Include only the ones with price above EMA, which are considered in uptrend.
        // Get the top 10 most liquid stocks with threshold of 1B to ensure the EMA validity from frequent market activities and trade liquidity to avoid extra slippage.
        return (from f in fundamental
            let avg = _state_data.GetOrAdd(f.Symbol, sym => new SelectionData(200))
            where avg.Update(f.EndTime, f.AdjustedPrice)
            where avg.Ema.IsReady && f.Price > avg.Ema.Current.Value && f.DollarVolume > 1000000000
            orderby f.DollarVolume descending
            select f.Symbol).Take(10);
    }
}

In this example, the SelectionData class group variables for the universe selection and updates the indicator of each asset. We highly recommend you follow this pattern to keep your algorithm tidy and bug free. The following snippet shows an example implementation of the SelectionData class, but you can make this whatever you need to store your custom universe filters. Note that the preceding SelectionData class uses a manual EMA indicator instead of the automatic version. For more information about universes that select assets based on indicators, see Indicator Universes. You need to use a SelectionData class instead of assigning the EMA to the Fundamental object because you can't create custom propertiesattributes on Fundamental objects like you can with Security objects.

Example 3: Take 10 stocks that are the furthest above their 10-day SMA of volume

The process to get the 10-day SMA stock volume is the same process as in Example 2. First, you should define a SelectionData class that performs the averaging. This class tracks the ratio of today's volume relative to historical volumes. You can use this ratio to select assets that are above their 10-day simple moving average and sort the results by the ones that have had the biggest jump since yesterday. 

public class HighRelativeVolumeUniverseAlgorithm : QCAlgorithm
{    
    // A dictionary to save the Volume SMA data and calculated fields for filtering.
    private ConcurrentDictionary<Symbol, SelectionData> _state_data = new ConcurrentDictionary<Symbol, SelectionData>();

    public override void Initialize()
    {
        // Add universe with custom selection rules for filtering.
        AddUniverse(FundamentalFilterFunction);
    }
    
    private IEnumerable<Symbol> FundamentalFilterFunction(IEnumerable<Fundamental> fundamental)
    {
        // ilter the ones with higher trading volume than SMA, indicating higher capital flow.
        // Get the top 10 highest relative volume stocks, since they have the highest capactity for scalp-trading or intra-day movement.
        return (from f in fundamental
            let avg = _state_data.GetOrAdd(f.Symbol, sym => new SelectionData(10))
            where avg.Update(f.EndTime, f.Volume)
            where avg.VolumeRatio > 1
            orderby avg.VolumeRatio descending
            select f.Symbol).Take(10);
    }
}

// Custom class to process SMA volume ratio information.
private class SelectionData
{
    public readonly Symbol Symbol;
    public readonly SimpleMovingAverage VolumeSma;
    public decimal VolumeRatio;

    public SelectionData(Symbol symbol, int period)
    {
        // Create SMA of volume to track the popularity of the stock.
        Symbol = symbol;
        VolumeSma = new SimpleMovingAverage(period);
    }
    
    public bool Update(DateTime time, decimal value)
    {
        // Update SMA with the updated data and calculate the relative volume position for filtering.
        var ready = VolumeSma.Update(time, value);
        VolumeRatio = value / VolumeSma.Current.Value;
        return ready;
    }
}
class HighRelativeVolumeUniverseAlgorithm(QCAlgorithm):
    # A dictionary to save the Volume SMA data and calculated fields for filtering.
    _state_data = { }

    def initialize(self) -> None:
        # Add universe with custom selection rules for filtering.
        self.add_universe(self._fundamental_filter_function)
    
    def _fundamental_filter_function(self, fundamental: List[Fundamental]) -> List[Symbol]:
        for f in fundamental:
            # To create and update Volume SMA indicators for each stocks for estimating their popularity.
            if f.symbol not in self._state_data:
                self._state_data[f.symbol] = SelectionData(f.symbol, 10)
            avg = self._state_data[f.symbol]
            avg.update(f.end_time, f.adjusted_price, f.dollar_volume)

        # Filter the ones with higher trading volume than SMA, indicating higher capital flow.
        values = [sd for sd in self._state_data.values() if sd.volume_ratio > 1]
            
        # Get the top 10 highest relative volume stocks, since they have the highest capactity for scalp-trading or intra-day movement.
        values.sort(key=lambda sd: sd.volume_ratio, reverse=True)[:10]

        return [ sd.symbol for sd in values ]
